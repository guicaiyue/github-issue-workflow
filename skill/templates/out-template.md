# `<stage-name>` 输出模板

## 该阶段完成报告应包含什么

- 输出要点 1
- 输出要点 2
- 输出要点 3
- 对下一阶段的建议及原因

## 建议字段

- `Current type: <stage-name>` 或 `Final type: <stage-name>`
- `Field A`
- `Field B`
- `Suggested next type`

## 使用说明

- 这里定义的是当前阶段写回 issue comment 时建议使用的输出结构。
- 输出模板应服务于 handoff、review 或 closeout，而不是重复输入模板。
- 如果当前阶段通常没有下游阶段，也应明确记录终态结论、证据与 follow-up 规则。
